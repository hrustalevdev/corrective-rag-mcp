# Выбор моделей — LLM и эмбеддинги

## LLM: почему llama3.1:8b

Первоначально использовались модели `qwen2.5:3b` и `phi3:latest` — лёгкие, не требуют мощного GPU.

**Проблема:** `gradeChunks` оценивал практически все чанки как `"no"` даже при очевидном тематическом совпадении. Граф уходил в retry-цикл, `ask_question` возвращал `chunks_used: 0, retries: 2`. Инструмент `find_relevant_docs` при этом находил отличные чанки — проблема именно в способности модели следовать бинарной инструкции с контекстом.

**Попытки исправить через промпты:** упрощение формулировки до «Reply with a single word: yes or no», явное указание критерия, структурирование контекста. Качество улучшилось незначительно — фундаментальное ограничение 3B-модели не преодолевается настройкой промптов.

**Решение:** переход на `llama3.1:8b`, запущенную на удалённом ПК (`http://192.168.1.10:11434`, RTX 3070 Ti 8 GB). Модель помещается в VRAM и справляется с задачами grading, broadenQuery и generateAnswer.

## Параметры Ollama — numCtx критичен

При переходе на `llama3.1:8b` вынесли три параметра в env:

| Параметр | Значение | Зачем |
|---|---|---|
| `OLLAMA_TEMPERATURE` | `0` | Детерминированные ответы |
| `OLLAMA_NUM_CTX` | `8192` | Дефолт llama3.1 = 2048 — не хватает для 5 чанков по 250 токенов + промпт |
| `OLLAMA_NUM_PREDICT` | `1024` | Ограничить длину ответа |

`numCtx` — самый важный параметр для RAG. При дефолте 2048 контекст `generateAnswer` (5 чанков × 250 токенов + инструкции) не вмещается и ответы обрезаются.

## Эмбеддинги: почему bge-m3

**`nomic-embed-text` (v1.5) — ограничения:**
- Обучена преимущественно на английских данных
- Кириллицу токенизирует неэффективно — каждый символ часто = отдельный токен
- На русском тексте заметная деградация качества
- Межязыковой поиск (русский запрос → английский документ) практически не работает

**Компенсирующий фактор:** гибридный поиск BM25 + vector через RRF-fusion частично скрывает слабость эмбеддингов — BM25 работает по ключевым словам независимо от языка модели.

**`bge-m3` — выбранная замена:**
- BAAI, специально обучена на 100+ языках включая русский
- 1024-мерные векторы (против 768 у nomic) — богаче семантическое пространство
- Контекст 8192 токенов — подходит для чанков 1000 символов без обрезки
- Стабильно в топ-3 MTEB Multilingual benchmark
- Доступна в Ollama: `ollama pull bge-m3`

**Важно при смене модели:** размерность меняется (768 → 1024), существующая коллекция ChromaDB становится несовместимой. Необходимо переиндексировать — ChromaDB выдаст ошибку измерений при попытке запросить старую коллекцию.
